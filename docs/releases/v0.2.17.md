# Weavatrix 0.2.17

0.2.17 is a self-audit trust and maintainability patch. It keeps the existing 38-tool surface,
offline defaults and MIT license while fixing false positives found by running Weavatrix 0.2.16
against both its own source and Weavatrix Hosted.

## Health and dependency precision

- Drizzle configuration files now root their configured schema modules in production reachability.
  Framework-owned schema exports no longer appear as orphaned or test-only Health actions.
- Scoped typosquat review compares scoped packages only with compatible scoped targets. Legitimate
  packages such as `@playwright/test` are not measured against unrelated unscoped names.
- Obsolete Poetry, uv and dist-info parser exports that were used only by tests were removed. The
  supported Python supply-chain path remains covered by the live lockfile and installed-package
  parsers.

## Duplicate-analysis trust

- Executable regular-expression literals now contribute one-way equality anchors to clone
  fingerprints. Their source text is not emitted, but unrelated regex-driven validation and
  normalization pipelines no longer become perfect renamed clones merely because their call shape
  is similar.
- Shared owners now cover bounded integer options, Go/Cargo dependency scopes, graph node IDs,
  architecture-contract lookup, safe repository reads and Hosted canonicalization/Blockly event
  handling.
- Final dogfood leaves the core with one intentional production clone group: the common static
  shell of the privacy and security legal pages. Hosted's remaining groups are declarative schema,
  Blockly definition or boundary-model shapes rather than repeated operational logic.

## Product boundary

- The accepted 0.3 ADR keeps `weavatrix` public under MIT as the offline graph/analysis engine.
- A separate public-source `weavatrix-online` connector will own all outbound HTTP capabilities.
  Its permission-required license is a release gate and remains unpublished until approved.
- Managed Cloud and licensed self-hosted Enterprise remain separate private deliverables sharing a
  versioned Hosted core instead of a permanent source fork.

## Verification

The release is gated by the full Node suite, exact LSP dogfooding, release-manifest verification,
npm/MCPB pack and runtime checks, and the Hosted typecheck/build/lint/test suite. The final core
Health audit reports zero critical/high/medium/low findings, zero runtime cycles, complete supported
dependency verification and a partial semantic overlay with exact LSP evidence.

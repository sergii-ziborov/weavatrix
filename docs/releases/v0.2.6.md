# Weavatrix 0.2.6

## Compact context and TypeScript identity patch

This release turns precise symbol inspection into a smaller agent workset and closes two remaining
TypeScript graph-fidelity gaps without adding a runtime dependency or changing the local-first
security boundary.

### Compact symbol context

- New `context_bundle` combines the selected definition, exact-reference evidence, grouped inbound
  and outbound graph containers, exact re-export locations, and bounded definition/caller excerpts.
- Related containers, re-export sites, context lines, and source files all have explicit hard caps.
  Text mode stays concise; JSON exposes the stable structured result and completeness state.
- The tool reuses `inspect_symbol` point precision instead of creating a second semantic engine.
  `GRAPH_ONLY`, `PARTIAL`, `UNAVAILABLE`, and capped evidence remain visible.

### Exact re-export occurrences

- JavaScript/TypeScript named, aliased, namespace, type-only, and star re-exports retain their exact
  file, line, specifier, exported/imported name, target file, and resolved declaration origin.
- `context_bundle` propagates the selected declaration through bounded barrel chains, including
  `export *`, while suppressing private declarations that are not part of the source module's export
  surface.
- Graph metadata versions this contract as `reExportOccurrencesV: 1`; `graph_stats` reports its
  availability and exact-site count.

### TypeScript type/value identities

- Interfaces and type aliases are first-class type-space nodes instead of being merged into a
  same-named runtime declaration.
- Runtime variables/functions/methods remain value-space nodes. Classes and enums are explicitly
  marked `both` because TypeScript exposes them in both namespaces.
- Static type references and type-only imports/re-exports resolve only to compatible type-space or
  merged-space nodes. Runtime calls cannot bind to a type-only declaration.
- Symbol-space metadata is preserved by graph diffs and the allowlisted hosted sync node shape.

### Compatibility and safety

- The graph extractor schema is version 5. Graphs missing re-export-occurrence or symbol-space
  metadata are rebuilt before precision-sensitive use, including `open_repo build:false` probes.
- Exact occurrence records and context excerpts remain local. Source bodies, snippets, absolute host
  paths, environment values, credentials, and Git remotes are still excluded from sync payloads.
- CFG reachability, data flow, taint/security evidence, and interprocedural complexity remain outside
  the local 0.2.6 claims and are reserved for a separately versioned hosted analysis layer.

# Weavatrix 0.2.1 — bounded agent output

`0.2.1` is a focused reliability patch for the repository-intelligence workflows introduced in
`0.2.0`. It fixes an agent-context regression in `git_history`, removes build-artifact noise from
hub ranking, and makes text output genuinely compact.

## Critical fix: bounded Git-history evidence

- `git_history top_n=N` now caps every structured collection exposed through MCP: `fileChurn`,
  `hotspots`, observed coupling, expected test/source coupling, and hidden coupling.
- Structured JSON includes deterministic per-collection `total`, `returned`, and `truncated`
  metadata plus the effective limit, so machine consumers know when more local evidence exists.
- The complete internal analytics object remains available to local analysis code; only the MCP
  boundary is bounded.

## Cleaner production signals

- `god_nodes` excludes tests, e2e, generated/build output, mocks, stories, docs, benchmarks,
  temporary paths, and repository-classified exclusions by default. Pass
  `include_classified:true` for an explicit all-surface inspection.
- Conventional `dist`, `build`, `out`, `coverage`, and `.next` roots are classified as generated
  output, preventing minified bundles from outranking product symbols.
- `change_impact` labels test/e2e edits as `test-only`, preserves the underlying change
  classification as evidence, and keeps them out of the product blast-radius seed set.
- `prepare_change` without a saved architecture contract now returns bounded provisional
  no-regression budgets and separates product files from test-only files. The guidance is explicitly
  non-enforceable until an owner-approved contract is saved.
- A missing architecture contract no longer repeats the full starter in every architecture tool.
  The starter is available only from `get_architecture_contract output_format:"json"`; normal text
  results return a concise remediation path.

## Compact text, explicit JSON

- The default `output_format:"text"` now returns only the concise MCP TextContent summary. It no
  longer duplicates the complete structured envelope into an agent's context.
- `output_format:"json"` remains the explicit opt-in for the stable `weavatrix.tool.v1` envelope
  and machine workflows. This also keeps repeated architecture starter contracts and per-finding
  boilerplate out of normal text conversations.

## Better cross-repository route tracing

- `trace_api_contract` resolves bounded local string constants used as prefixes in template URLs,
  including chains such as `` `${API_ROOT}/query` ``. Truly dynamic segments remain uncertain.
- Its `path` filter accepts segment-aligned route fragments, so `/query` can select
  `/edgeAnalytics/query/...` without discarding the backend prefix.

## Upgrade

```sh
npx -y weavatrix@0.2.1 <repoRoot>
```

No graph migration or configuration change is required. The default profile remains local and
offline; network-capable tools are still explicit opt-ins.

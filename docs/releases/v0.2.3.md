# Weavatrix 0.2.3

## Real HTTP-wrapper patch

- Recognizes the common fixed-method transport pattern `api(axios.get, [url, options])` and its
  POST/PUT/PATCH/DELETE/HEAD/OPTIONS equivalents without evaluating source code.
- Keeps discovery conservative: the client, method, first transport argument and wrapper parameter
  must all be statically identifiable, and imported wrappers remain bounded to their reverse-import
  scope.
- Resolves an otherwise ambiguous backend handler name when exactly one matching symbol belongs to
  a module directly imported by the route file.

On the real `frontend` + `analytics` repositories used for post-publication verification, the same
source-free trace improved from 2 client calls and 0 matches to 406 calls and 44 matches. Twenty-nine
backend handler nodes received medium/high-confidence `NOT_DEAD_EXTERNAL_USE` evidence and became
eligible to suppress only their exact dead-code candidates. The bounded text response remained
approximately 6.3 KiB; incomplete affected-file traversal stayed explicitly `PARTIAL`.

This patch adds no dependency and preserves the 0.2.2 graph, sync, provenance and configuration
contracts.

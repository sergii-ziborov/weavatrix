# v0.1.5 (draft)

> Draft release notes only. The package version remains `0.1.4` until the hosted receiver and its
> privacy text are deployed and the end-to-end compatibility gate passes.

## Evidence sync, without source upload

`sync_graph` can now construct payload v3. It keeps the existing graph allowlist and adds a
deterministic `EvidenceSnapshot V1` derived locally from the active repository:

- folder modules and separately classified runtime, type-only and compile-only dependencies;
- runtime and compile-time cycles with member files and a representative path;
- violations of declared `.weavatrix-deps.json` boundaries with the exact dependency direction;
- health rollups and typed findings, without titles, details, snippets or arbitrary analyzer text;
- complexity hotspots against explicit LOC, cyclomatic and parameter thresholds;
- manifest/file stack identifiers;
- pinned package versions and direct package usage.

Each section reports `state`, `verdict` and bounded completeness counts. Missing OSV, malware,
coverage or architecture-style analysis stays unknown; it is never rendered as a clean zero.
Package evidence is explicitly `PARTIAL` because transitive lockfile parent edges are not available
yet. The existing broad infra detector is deliberately excluded until its import attribution is
made reliable.

## Contract and security

- V2 remains unchanged and can be selected explicitly with `payload_version: 2`.
- V3 is deterministic and recomputes a canonical SHA-256 after wire sanitization.
- V3 has no fields for source bodies, snippets, absolute host paths, environment values, credentials,
  Git remotes, timestamps or free-form analyzer errors; unknown fields are discarded and unsafe
  optional path metadata is omitted.
- V3 normalizes repository paths, validates symbol suffixes and `id/source_file` consistency, and
  rejects duplicate node IDs or dangling links.
- Client limits mirror hosted limits: 8 MiB, 25,000 nodes, 100,000 links and 50,000 external imports.
- A stale graph is refused before source-derived evidence is built.
- Network requests have a bounded timeout and never silently downgrade from V3 to V2.

## Signal quality

- `no-tests` and clone scanning now recognize Cypress, Playwright, `test-e2e`, acceptance and
  integration roots. A repository-root `.weavatrixignore` gives graph, audit and duplicate scans one
  shared tracked-file exclusion list.
- `query_graph` preserves directed import output, ranks architectural entry points for broad
  bootstrap/auth/router questions, and accepts exact `seed_files`.
- Cycle findings include a closed representative route; huge cycles keep a bounded head/tail with
  the omitted member count. The known `query-string` package no longer triggers a `querystring`
  typosquat warning.
- Default registrations are repository-pinned. Retargeting now requires an explicit `retarget`
  capability, and target-mutating calls are serialized while ordinary calls retain their starting
  graph/root snapshot.
- Dependency findings state their reason. Sass/Less/Stylus compilers are recognized from source
  extensions, while CSS-only and type-only imports remain high-confidence direct-dependency evidence.
- Identical graph-staleness warnings are throttled for five minutes; changed stale state appears
  immediately and `graph_stats` always reports freshness.

## Release gate

Do not publish this package version until the hosted receiver accepts and re-sanitizes V3, checks
the payload-version header against the body, canonicalizes every nested evidence array, preserves
V1/V2 regressions, and the public privacy policy describes local evidence derivation plus the
repository display label sent by sync.

# Weavatrix 0.2.5

## Exact symbol and graph-fidelity patch

This release closes the main precision gaps found by the 0.2.4 competitive review while keeping
Weavatrix local-first and read-only.

### Exact, bounded symbol inspection

- New `inspect_symbol` combines one declaration, exact TypeScript/JavaScript reference occurrences,
  logical reference containers, direct graph evidence, bounded reverse impact, and small source
  excerpts in one call.
- Explicit point queries bypass the graph-wide LSP selection cap for the requested symbol. They use a
  separate revision/config-fingerprinted `precision-symbols.json` LRU and never overwrite the broad
  `precision.json` overlay.
- Ambiguous labels fail closed and require an exact node ID. Time, occurrence, container, source-size,
  cache-size, and project-safety limits remain explicit in the result.

### Better static graph fidelity

- JavaScript/TypeScript default-object facades such as `export default { getSchema, save: persist }`
  now resolve member calls to the real local declaration.
- `get_dependents` stays symbol-precise by default. The old containing-file expansion is available
  only through `include_container_importers:true` for deliberately conservative module impact.
- Python methods retain class ownership. Typed parameters, direct constructor assignments,
  `self`/`cls`, module member calls, aliases, and unambiguous `from x import *` names (including static
  `__all__`) resolve to the correct callable. Ambiguous or dynamic receiver evidence is not guessed.

### Health signal fixes

- `run_audit` now always states dependency-manifest completeness and the checked declared/import
  counts, including explicit `unused 0` and `missing 0` results.
- Production declarations whose only consumers are tests/e2e appear as medium-confidence
  `test-only-*` review candidates and `test-only-symbol` audit findings.
- `find_duplicates` can scan high-confidence small clones down to 12 tokens when `min_tokens` is
  lowered. Small fragments require stricter similarity and bounded fingerprint evidence; the normal
  default remains 50 tokens.

### Local hot-path review

- New offline `hot_path_review` ranks bounded production symbols from parser-derived local time,
  memory, cyclomatic and call-site evidence without claiming profiler measurements.
- Complexity schema v2 records line-addressable allocations, copies, linear scans, sorts and direct
  recursion inside iteration. The tool reports these local syntax signals separately from graph
  fan-in/fan-out risk.
- Existing coverage reports are mapped onto symbol ranges. Without measured coverage, static test
  reachability is labelled `actualCoverage: NOT_AVAILABLE`, never presented as executed coverage.
- Thresholds, path scope, production/test classification, minimum score and `top_n` are bounded and
  configurable. Interprocedural Big-O, recursion bounds, CFG reachability, dead stores and taint flow
  remain explicit non-claims.

### Compatibility and safety

- The graph extractor schema is version 4; older graphs are rebuilt before precision-sensitive use.
- All new analysis is local. No repository body or point-query evidence is transmitted, and no
  repository language-server binary is executed.

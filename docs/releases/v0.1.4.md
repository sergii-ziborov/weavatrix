# Weavatrix 0.1.4 — Rust graph depth and Java OOP recovery

This release keeps the 23-tool MCP surface intact while making Rust and Java graphs substantially
more useful and reducing false runtime findings in mixed-language repositories.

## Highlights

### Rust module graphs and HTTP endpoints

- `mod`, `use` and `pub use` resolve across crate roots, nested modules, `mod.rs`, aliases, grouped
  imports, wildcard imports and `#[path]` overrides.
- Rust module/use edges are marked `compileOnly`; they enrich `module_map` without creating fake
  runtime cycles or boundary violations, and compile-time dependents are labeled instead of being
  promoted to runtime impact.
- `list_endpoints` recognizes axum route builders/chains and actix-web route macros, resources and
  method builders.
- Cargo crate roots and `build.rs` are framework entry points rather than dead/orphan candidates.

Measured on `AI-Dev-System`: 324 Rust import/re-export edges are compile-only, 49 cross-module
compile-only dependencies are visible, all 11 axum/actix endpoints are found, and the previous
58/10/3-file runtime-cycle alarms are now three `info` compile-time coupling groups with zero
runtime cycles.

### Java OOP relationships and package territories

- Classes, interfaces, enums, records and annotations retain their symbol kind; methods and
  constructors retain their declaring type and visibility.
- Class-to-method/constructor ownership is explicit but structural: it stays visible in navigation
  and `god_nodes` while never counting as runtime usage, blast radius or proof that dead code is live.
- Internal `extends`, interface `extends`, `implements`, same-package/imported types and resolvable
  type references link to real declarations.
- Java imports are compile-only. Calls, references and heritage contribute impact signal.
- Standard Maven/Gradle `src/main/java` and `src/test/java` trees use deterministic package-aware
  communities instead of collapsing into one top-level `src` blob.
- External types do not become synthetic placeholder nodes, and an external import cannot bind to
  an unrelated same-basename project class.

Measured on `warroom`: 7,143 nodes / 17,529 links, including 3,818 method-ownership edges, 3,808
declaration-backed references, 54 `inherits`, 25 `implements` and 1,701 compile-only imports. The
graph now has 33 named package territories instead of four coarse buckets, with zero runtime import
cycles. These territories are deterministic package grouping, not connectivity-clustering parity
with legacy Graphify.

### Honest compile-time semantics

- Graph edge metadata advances to `edgeTypesV: 2`.
- `module_map`, `god_nodes`, `get_dependents`, `change_impact`, `graph_diff` and `run_audit` separate
  runtime, TypeScript type-only and language compile-only edges consistently.
- A schema-upgrade rebuild establishes a new baseline; newly extractable Rust/Java edges are not
  mislabeled as code-caused architecture drift.
- Sync payload v2 preserves optional `typeOnly` and `compileOnly` link metadata. Source bodies and
  absolute repository paths remain excluded.

### Lower audit noise without broad whitelists

- Generated NAPI-RS loaders suppress only known platform/architecture packages and generated local
  `.node` fallbacks. Same-prefix packages such as `foo-utils` remain visible as missing dependencies,
  and npm scopes must match.
- Conventional template/example/sample/fixture/snippet roots are recognized conservatively.
  Custom roots can be declared with `nonRuntimeRoots` or `templateRoots` in
  `.weavatrix-deps.json`; import edges/cycles/boundaries remain visible.
- Missing/unresolved dependency findings are suppressed only when every use is inside a declared
  non-runtime root.
- Framework runtime peers such as Next.js + `react-dom` are retained without a repo-wide whitelist.
- Runtime summaries and hotspot ranking ignore structural ownership edges while still reporting
  Java method ownership separately.

## Cross-language regression check

- TypeScript (`ivrit-inbox`): typed imports, re-exports and references remain ahead of the saved
  Repo Lens graph; ignored packaged output no longer reappears.
- JavaScript (`analytics`): mature JS extraction remains intact; the saved comparator graph predates
  the repository HEAD, so raw totals are not treated as a regression signal.
- Go (`bgp-speaker`): all 1,165 edges match the saved Repo Lens graph exactly.
- Python (`automation`): the rebuilt Weavatrix graph is unchanged at 7,381 nodes / 19,213 links,
  including all 430 inheritance edges.
- Rust (`AI-Dev-System`): Rust symbols/calls remain ahead while module/use paths and endpoints fill
  the previously missing surfaces.
- Java (`warroom`): OOP relations and package territories are restored without synthetic metric
  padding. Cross-class receiver call inference remains unchanged and is future work.

## Upgrade

```sh
npx -y weavatrix@0.1.4 <repoRoot>
```

Graphs built before 0.1.4 need one rebuild for edge metadata v2. A normal `open_repo` call performs
that upgrade automatically; `build:false` remains a non-building probe and refuses legacy graphs.
Call `rebuild_graph` once for the currently active repository before relying on compile-only
classifications or calling `sync_graph`.

The local MCP remains offline by default. `refresh_advisories` and `sync_graph` still require the
explicit `online` capability group and an individual tool call. Requires Node.js 18 or newer.

# Weavatrix 0.2.2

## Regression foundation

- Adds a permanent TS/JS/Python/Go/Java/Rust golden benchmark with graph-byte, cold-latency and
  source-free report budgets.
- Exercises the real MCP lifecycle across cold full build, incremental refresh, unchanged reuse and
  reconnect while verifying active repository/revision stability and bounded text output.
- Adds portable 0.2.1 baselines for five available real repositories, explicit
  `MISSING`/`UNBASELINED`/`STALE` states, and a strict all-six release command.
- Gates representative Java class/interface/member/OOP/endpoint signals and Rust module/endpoint
  signals. Missing Rust source is reported as `SOURCE_CHECKOUT_MISSING`, never as a green check.
- Covers framework peers, build tools, convention consumers and generated/e2e classification with a
  negative control that prevents blanket suppression.

## Edge evidence

- Adds versioned edge provenance: `EXACT_LSP`, `EXTRACTED`, `RESOLVED`, `INFERRED`, and `CONFLICT`.
  The current static builder emits complete extracted/resolved/inferred coverage; exact LSP and
  conflict states are reserved for the optional local precision bridge.
- Keeps legacy `confidence` and the payload-v2 wire shape compatible. Payload v3 allowlists
  provenance and requires a current provenance schema.
- `graph_stats`, golden cases and real-repository gates expose provenance completeness.

## Cross-repository contracts

- Recognizes configured and conservatively auto-discovered HTTP wrappers, including bounded dynamic
  URL prefixes.
- Resolves backend handlers where unambiguous and reports `NOT_DEAD_EXTERNAL_USE`,
  `POSSIBLE_EXTERNAL_USE`, or `UNKNOWN`; only medium/high external evidence on a resolved handler can
  suppress a dead-method candidate.

This release adds no mandatory runtime dependency.

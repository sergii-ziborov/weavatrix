# Weavatrix 0.2.0 — evidence-driven repository intelligence

`0.2.0` turns Weavatrix from a collection of graph queries into a review workflow: classify the
actual changed symbols, state a risk verdict, compare against an immutable Git baseline, trace HTTP
contracts across repositories, and verify an owner-approved target architecture. The local-first
default remains offline.

## Review the change, not the whole repository

- `change_impact` parses zero-context Git hunks and classifies every affected symbol as `added`,
  `removed`, `signature-changed`, `body-changed`, `metadata-only`, or `unknown`.
- Pure additions no longer inherit every legacy importer of a busy barrel/API file. They return a
  low-risk additive verdict and a zero legacy blast radius unless explicitly requested otherwise.
- Signature removals/changes seed the exact symbol and relevant module surface; output begins with a
  bounded `LOW`, `MEDIUM`, or `HIGH` conclusion and the evidence behind it.
- Static test reachability is reported when no coverage artifact exists. It names graph paths from
  indexed tests while explicitly saying that reachability is not executed line/branch coverage.
- `run_audit` can compare an immutable `base_ref` and report genuinely new, existing, and fixed
  deterministic debt. A changed-file-only scope is labeled as scope, never falsely called new debt.
- New `find_dead_code` turns symbol-level liveness into a bounded review queue for files, functions,
  methods and declarations. It returns confidence, reason and source-free evidence; tests/generated
  surfaces are excluded by default, while public API, framework, dynamic-loading and reflection risk
  is explicit and low-confidence. Its verdict is always review-required, never auto-delete.

## Immutable graph diffs and automatic freshness

- `graph_diff base_ref=HEAD~1` resolves the ref to a commit, exports it to a private temporary
  checkout, builds the baseline graph, compares it with the current graph, and removes the checkout.
  Graph artifacts do not need to be committed.
- Graph and health calls automatically reconcile the saved graph before answering. A body-only edit
  reparses the changed JS/TS file plus bounded reverse importers; unchanged files retain their full
  symbol/export tables.
- Consecutive reads use a short process-local freshness debounce plus a Git dirty-content probe, so
  auto-sync does not rescan every clean source file before every tool call. Canonical graph writes and
  repository-registry updates are cross-process locked and atomically replaced.
- Add/delete, barrel, alias, manifest, ignore, export-surface, or unsafe merge cases fall back to a
  full rebuild. Structured results state `full`, `incremental`, or `none`, revision, and changed-file
  count.
- Re-export barrels are transparent for semantic dependencies: proxy edges preserve physical import
  evidence while god nodes, dependents, queries, and module pressure point to the originating module.

## Cross-repository HTTP contracts

- New `trace_api_contract` selects repositories only from the local stable registry by UUID or exact
  label; arbitrary filesystem paths are not accepted.
- It joins backend endpoints to static frontend/client calls, follows client importers to affected
  screens, and supports method/path/backend-changed-file filters plus bounded traversal/output caps.
- The result is verdict-first and source-free: endpoint, matched callsites, affected files/screens,
  confidence, completeness, and unresolved dynamic routes.

## Architecture as an executable direction

- `get_architecture_contract`, `prepare_change`, and `verify_architecture` now provide a usable
  starter/remediation path instead of ending at `NOT_CONFIGURED`.
- Contracts define components, allowed/forbidden dependency directions, technology rules, exceptions,
  complexity budgets, module-size/cohesion/boundary-pressure fitness functions, and enforcement mode.
- Ratchet mode separates new violations from accepted legacy debt and reports fixed fingerprints.
- `pull_architecture_contract` explicitly downloads the owner-approved hosted contract using the
  stable repository UUID and bearer authentication, validates it, and caches it beside the local
  graph. No source, symbol, or absolute repository path is sent.

## Git history and package evidence

- `git_history` combines churn with graph connectivity and reports co-change coupling using support,
  confidence, lift, and Jaccard evidence. Hidden coupling is called out only when files repeatedly
  change together without a static dependency edge.
- Evidence sync includes a bounded package-lock v2/v3 dependency graph: direct and transitive package
  nodes, runtime/dev/optional/peer edges, nested resolution, and explicit unresolved/local/optional
  completeness counts.
- Evidence sync also includes bounded, stable clone groups and conservative same-name divergence
  candidates. Only repository-relative coordinates, similarity/fingerprint metadata, and review
  evidence are sent—never source bodies or snippets.
- Package identities are versioned and path-disambiguated without sending install paths, integrity
  hashes, resolved URLs, source text, environment values, credentials, or Git remotes.
- OSV findings are joined with direct product import reachability. Unused/unobserved vulnerable
  packages are not silently called safe; their priority and confidence are lowered with the reason.

## Signal quality and repository control

- `.weavatrix.json` classifies tests, e2e, generated, mocks, stories, docs, benchmarks, temporary
  `__temp` roots, and explicit exclusions; narrow `classify.product` patterns can opt a verified
  production benchmark back into review;
  `.weavatrixignore` remains the shared tracked-file exclusion layer.
- Cypress/Playwright/test-e2e noise is excluded when tests are disabled. Clone output suppresses
  classified non-product fragments unless explicitly requested.
- `query_graph` supports pinned `seed_files`, prioritizes architectural entry points for broad
  bootstrap/router/auth questions, and preserves true import direction.
- The graph store and repository registry are global and collision-safe under `~/.weavatrix`, so
  moving between unrelated parent directories does not create disconnected stores.
- Structured MCP results use `weavatrix.tool.v1`; `output_format=json` returns the same stable
  envelope without requiring agents to parse prose.

## One package, explicit security profiles

No network capability is exposed by the default profile, and enabling a profile never performs a
request by itself.

| Profile | Repository switching | Cross-repo graph reads | OSV | Hosted sync / contract pull |
|---|---:|---:|---:|---:|
| `offline` (default) | explicit local `open_repo` | explicit `trace_api_contract` | no | no |
| `pinned` | no | no | no | no |
| `osv` | explicit local `open_repo` | explicit tool call | explicit tool call | no |
| `hosted` / `full` | explicit local `open_repo` | explicit tool call | explicit tool call | explicit tool call |

The old `online` capability remains a compatibility alias for `advisories,hosted`. This is still one
npm package and one binary; the profile only controls which MCP tools are registered.

## Upgrade

```sh
npx -y weavatrix@0.2.0 <repoRoot>
```

Choose `pinned`, `osv`, or `hosted` as the final argument when needed. Existing graph files are
reconciled automatically; use `rebuild_graph` once if an older graph cannot satisfy the current
edge/schema contract. Requires Node.js 18 or newer.

Before publishing, the tagged source is tested, audited, packed, and used to build the MCPB artifact.
GitHub Actions publishes npm with trusted publishing and SLSA provenance, attests the MCPB, verifies
the registry `gitHead`, publishes MCP Registry metadata, and creates an immutable GitHub release.

Privacy Policy

Effective: July 15, 2026

The short version

Weavatrix is a local-first tool. It runs entirely on your machine, collects no telemetry, requires no account, and sends no data to us — we operate no servers that receive anything from the software.

What the software processes

The Weavatrix MCP server analyzes source code repositories on your own computer. All analysis (graph building, audits, clone detection, coverage mapping) happens locally. Derived data — the code graph — is written next to your repository (weavatrix-graphs/) and an advisory cache to ~/.weavatrix/. Nothing is uploaded.

The only network-capable features (opt-in)

  • refresh_advisories — when you explicitly invoke it, the names and versions of packages pinned in your lockfiles are sent to OSV.dev (a Google-run public vulnerability database) to fetch security advisories. No source code is transmitted.
  • sync_graph — disabled unless you configure your own endpoint via the WEAVATRIX_SYNC_URL environment variable. When you do, the graph (file paths, symbol names, edges — never file contents) is sent to the endpoint you chose.

Both tools live in an online capability group that can be excluded entirely at registration time.

This website

weavatrix.com is a static page served by Cloudflare. We run no analytics and set no cookies. Cloudflare may process standard request logs (IP address, user agent) to serve and protect the site, per Cloudflare's privacy policy.

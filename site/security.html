Security

Current public release: 0.2.19

Suspected vulnerability? Please do not open a public issue. Use GitHub's private vulnerability reporting.

Supported versions

Security fixes are provided for the latest 0.2.19.x release. Users on 0.2.18 or older should upgrade before reporting an issue that may already be resolved.

Default boundary

The default offline profile exposes no HTTP tools. Repository reads are constrained to the active repository after canonical-path and symlink/junction checks. The stricter pinned profile also removes repository switching and cross-repository graph reads.

Optional network tools

  • refresh_advisories sends pinned package names and versions to OSV.dev only when called.
  • pull_architecture_contract sends only an opaque repository UUID to the configured authenticated endpoint.
  • preview_sync has no network path and produces the destination/exact-payload approval artifact. sync_graph sends only with both dry_run:false and its matching short-lived confirmation token. HTTPS is required outside loopback.

The graph-sync contract excludes source bodies, snippets, absolute host paths, environment values, credentials and Git remotes. Unknown payload fields are discarded.

Full policy

Process isolation, optional test execution, payload limits and reporting details are documented in the repository's complete security policy. See also the privacy policy and license.
